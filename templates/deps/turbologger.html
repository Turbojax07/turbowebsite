{% extends 'base/inline_md.html' %} {% block title %}TurboLogger{% endblock %} {% block header %}TurboLogger{% endblock %} {% block md %} # TurboLogger ## Links: - Vendordep URL: [https://turbojax.org/static/files/TurboLogger.json](https://turbojax.org/static/files/TurboLogger.json) - Javadocs: [https://turbojax.org/static/javadocs/turbologger/index.html](https://turbojax.org/static/javadocs/turbologger/index.html) - Github: [https://git.turbojax.org/TurboLogger](https://git.turbojax.org/TurboLogger) ## Usage: - Just put `import org.turbojax.TurboLogger` at the top of your file and log away! - All of the logging functions are static, so no need to make an instance of them. - At the moment, TurboLogger does not support logging units, so convert them to a number before logging, and handle recieving them accordingly. Example usage: ``` import org.turbojax.TurboLogger; ``` ## Aliases - Aliases are essentially shorter forms of keys. Rather than referencing "/motors/motor1/outputs/voltage" every time you want to read the voltage, you can make an alias called "m1voltage" and read that instead. - To make an alias, you can use `TurboLogger.addAliases()`. The first parameter is the key, and the next is an array of the aliases. - Example: ``` TurboLogger.addAliases("/motors/motor1/outputs/voltage", "m1voltage"); TurboLogger.addAliases("/motors/motor1/outputs/voltage", "m1voltage", "m1volts"); TurboLogger.addAliases("/motors/motor1/outputs/voltage", {"m1voltage", "m1volts"}); ``` # I am still working on this page!!! ## In the meantime, here are some rough descriptions of the functions. ## Quick TurboLogger walkthrough: ### Functions of TurboLogger `TurboLogger.enableDataLogs(String path)` → Allows logging to datalogs. The string is the filepath to create the logfile at. `TurboLogger.disableDataLogs()` → Disables datalog logging. `TurboLogger.log(String key, value, String... aliases)` → Logs the value to NetworkTables under the key parameter. The aliases vararg allows you to define aliases when you push a value without needing to run `TurboLogger.addAliases()`. Supports logging of all primitive data types, Strings, StructSerializable objects, and arrays of each of them. Returns nothing and marks the value as unread. `TurboLogger.get(String key, defaultValue)` → Returns an object/primitive that matches the type of the defaultValue. (It's why the function can be called simply "get" over "getBoolean" and others.) Marks the value as read. Supports all the same classes that the log function does. `TurboLogger.addAliases(String key, String... aliases)` → Registers all of the strings in the aliases array as references to the key. Aliases can be used in place of the key in `TurboLogger.get()`, `TurboLogger.log()`, and `TurboLogger.hasChanged()`. However, they do not mark other aliases as read when they are read. `TurboLogger.hasChanged(String key)` → Gets if the value of the key has changed. This returns true if the user has logged to the key since the last read, or if the variable changes in NetworkTables. `TurboLogger.remove(String key)` → Removes a key and all of its aliases from TurboLogger. ### Other functions (should've made them private tbh) `TurboLogger.pubSubTypeMismatch(String key, String type, boolean isPub)` → Utility function to help with logging internal errors. `TurboLogger.noSub(String key)` → Utility function to help with logging internal errors. `TurboLogger.structTypeMismatch(String key, String desiredType, String realType)` → Utility function to help with logging internal errors. ### Quick Examples: ``` // Logs the string "Value logged" to the "String/1" position in NetworkTables // Also creates an alias for "String/1" called "alias1" TurboLogger.log("String/1", "Value logged", "alias1"); // N/A // Gets the string from the alias. The default value is "DefaultVal" // Since we already registered "alias1" as an alias to "String/1", this would return "Value logged". TurboLogger.get("alias1", "DefaultVal"); // "Value logged" // Checks if the value was changed since we last read it. This will return "false". TurboLogger.hasChanged("alias1"); // false // Logging the string "New Value" using the alias. TurboLogger.log("alias1", "New Value"); // N/A // Now the hasChanged will return true for both the key and alias. TurboLogger.hasChanged("String/1"); // true TurboLogger.hasChanged("alias1"); // true // However if we read the value using the alias... TurboLogger.get("alias1", "DefaultVal"); // "New Value" // Then the alias will return false, but the key will remain true. // This functionality is included so that you can have multiple aliases listening to one value and allow each of them to update. (Like 4 PID controllers using the same kP, kI, and kD values). TurboLogger.hasChanged("String/1"); // true TurboLogger.hasChanged("alias1"); // false // We also have the addAliases() function, which allows us to create aliases without logging a value. TurboLogger.addAliases("String/1", "alias2", "alias3"); // N/A // Lastly we have the remove() function, which quite simply removes a key and its aliases from the logger. TurboLogger.remove("alias1"); // N/A ``` {% endblock %}
